# 外贸跟单记录工具

一个纯前端的本地记录工具，用来管理外贸订单进度、问题记录、下一步动作和工作日志自动更新。

## 使用方式

直接打开 `index.html` 即可使用。当前我也启动了本机预览服务：

```text
http://127.0.0.1:5179/index.html
```

数据保存在浏览器本地存储中，不会上传到服务器。换浏览器或清理浏览器数据前，建议先点左侧的导出按钮备份 JSON 文件。

## 主要功能

- 新建、编辑、删除订单
- 记录客户、国家、产品、数量、金额、交期、当前阶段、订单状态、下一步动作
- 添加、解决、删除订单问题
- 为每单登记联系方式，支持标记工厂、货主、货代或其他联系人，并保存姓名、带区号电话和标准格式邮箱
- 用链路节点查看订单从询盘到完成的进度
- 悬浮链路节点查看该阶段详情、问题和提醒
- 点击链路节点编辑阶段备注和提醒月日、时间；手动填写时不需要年份
- 上传或粘贴工作日志、订单表和资料文件，自动识别订单号、货物信息、重量、品类、联系人姓名、带区号电话、邮箱、进度阶段、风险状态、问题类型、下一步动作和提醒时间
- 支持按关键词和状态筛选订单
- 支持 JSON 导入和导出

## 工作日志示例

```text
PI-2026-001 今天客户确认打样，发现包装标签尺寸不符，下一步明天发新版唛头给客户确认。
PI-2026-002 工厂反馈大货生产延期，有交期风险，需要今天通知客户。
```

先点击“分析日志”查看识别结果，确认后点击“应用更新”。日志里出现“今天、明天、后天、5月15日、2026-05-15、下午3点”等日期和时间表达时，会自动写入对应节点提醒。

## 文件识别

点击“上传文件”会弹出系统文件选择器。支持上传 `txt`、`md`、`csv`、`json`、`xlsx`、`xls`、`pdf`、`docx`。Excel/CSV 如果包含表头，例如“订单号、客户、产品、重量、品类、姓名、电话、邮箱”，工具会按列提取并更新对应订单。PDF 和 DOCX 会先抽取正文文字，再按关键词识别信息。

## 提醒方式

提醒到点后会在网页内全屏弹窗，并可延后 10 分钟。网页打开时还会尝试使用浏览器通知，适合页面开着但不在前台的情况。

如果网页关闭后仍要提醒，需要把订单和提醒同步到后端，并由后端定时发出通知。免费可行路线建议优先做邮箱提醒：Supabase 免费层可保存提醒并定时执行任务，邮件可接 Resend 或类似邮件服务的免费额度。微信提醒通常需要公众号或企业微信接口，配置成本更高。

本项目已预留 Supabase + Resend 邮箱提醒后端代码，见 `supabase/README.md`。配置后，在网页“邮箱提醒”里填写接收邮箱、Supabase 函数地址和 anon key，点击“同步邮箱提醒”即可。
